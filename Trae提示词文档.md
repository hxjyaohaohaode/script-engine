# Trae提示词文档 — 互动影游剧本智能体集群

> 重要说明：本文档记录的是项目早期的提示词驱动搭建流程，包含若干历史路径与旧接口。
> 当前系统已经落地，实际实现以 `script-engine/README.md`、`script-engine/backend/api/*.py`、`script-engine/frontend/src/api/client.ts` 为准。
> 如本文出现 `backend/agents/*`、旧场景生成接口或旧模型命名，请视为历史内容，不应继续按此创建新代码。
>
> 使用方法：按顺序将以下提示词复制粘贴发给Trae，每完成一步再发下一步。
> 同时将以下两个文件作为附件上传给Trae：
> 1. `互动影游剧本智能体集群-完整技术方案.md`（系统设计）
> 2. `互动影游剧本智能体集群-开发规格书.md`（实现规格）

## 提示词 1：项目初始化

```
我要搭建一个互动影游剧本智能体集群的Web应用。

请先读一下我上传的两个文档（完整技术方案 + 开发规格书），理解整个系统的设计。

然后帮我做以下事情：

1. 创建项目文件夹 `script-engine/`，结构如下：
   - backend/ （Python FastAPI后端）
   - frontend/ （React + TypeScript + Tailwind前端）

2. 后端初始化：
   - 创建 requirements.txt（fastapi, uvicorn, sqlalchemy, asyncpg, alembic, pydantic, celery, redis, httpx, python-dotenv, gitpython）
   - 创建 main.py（FastAPI入口，配置CORS，挂载所有路由）
   - 创建 config.py（从环境变量读取数据库URL、Redis URL、API Key等）
   - 创建 database.py（SQLAlchemy异步引擎+会话管理）

3. 前端初始化：
   - 使用 Vite + React + TypeScript
   - 安装 tailwindcss、antd、react-router-dom、@tanstack/react-query、zustand、recharts、d3
   - 配置 tailwind.config.js
   - 创建基础路由结构（/、/world、/characters、/foreshadows、/chapters、/scenes、/review、/emotion-curve、/export）

4. 创建 docker-compose.yml（本地开发用，包含 PostgreSQL + Redis）

5. 创建 render.yaml（部署配置，包含 Web Service + Worker + PostgreSQL + Redis）

6. 创建 .env.example 文件

请确保代码能跑起来，后端能启动，前端能显示基础页面。
```

## 提示词 6：场景工作台（核心页面）

```
请创建场景工作台页面 /scenes，这是整个系统最核心的页面。

布局：左侧场景列表（25%宽度）+ 右侧场景编辑器（75%宽度）

左侧场景列表：
- 按章节分组，章节可折叠展开
- 每个场景显示：场景编号 + 状态图标 + 情感强度条
- 状态图标：✅已定稿 🔵进行中 ⬜待创作 🔴审计未通过 ⭐哇塞场景
- 点击场景切换右侧编辑器内容
- 章节标题旁显示本章完成进度
- [+ 新建场景] 按钮

右侧场景编辑器，分为以下区域：

A. 基本信息栏（顶部）：
   - 场景编号、场景类型（下拉选择）、地点、天气
   - 情感目标滑块（0-10，颜色渐变条）
   - 出场角色（多选，每个角色显示当前情感状态）
   - 是否哇塞场景（开关）
   - 状态标签（草稿/审计中/已封驳/已定稿）

B. 内容区（标签页切换）：
   - 标签1「场景描述」：富文本编辑框，支持长文本
   - 标签2「对白」：结构化对白编辑器
     - 每条对白：角色选择 + 对白文本 + 潜台词（可选）
     - 可增删对白条目
     - 对白顺序可拖拽调整
   - 标签3「动作指示」：文本列表，可增删
   - 标签4「选择节点」：
     - 每个选项：选项文本 + 后果描述 + 跳转场景 + 是否隐藏
     - 隐藏选项需填写前置条件
     - [+ 添加选项] 按钮

C. 伏笔操作区（右侧边栏）：
   - 显示本场景需操作的伏笔列表（从伏笔任务读取）
   - 每个伏笔：伏笔名称 + 操作类型（植入/强化/回收）+ 内容描述
   - [+ 添加伏笔操作] 按钮
   - 未完成的伏笔操作红色边框提示

D. 因果链区（可折叠）：
   - 前置条件（多行文本）
   - 催化剂
   - 直接结果
   - 间接结果
   - 远期结果

E. 操作栏（底部固定）：
   - [AI生成] 按钮：调用创作Agent生成场景（显示加载动画+预计时间）
   - [提交审计] 按钮：调用审计Agent审核
   - [人工编辑] 按钮：切换到手动编辑模式
   - [定稿] 按钮：写入Layer 1/2/5（需二次确认）
   - [重新生成] 按钮

F. 审计结果区（底部可展开）：
   - 程序化检测结果（6项，每项显示✅或❌+具体说明）
   - LLM审计结果（6项，每项显示评分+说明）
   - 综合判定：通过（绿色）/ 封驳（红色）+ 问题列表 + 修改建议
   - 历次审计记录（可折叠查看）

G. 版本历史（右下角）：
   - 版本列表（v1, v2, v3...）
   - 点击可查看/恢复历史版本
   - 每个版本显示修改时间和修改原因

边界情况：
- AI生成中：显示进度条+取消按钮，禁用其他操作
- 审计封驳：审计结果区红色高亮
- 连续3次封驳：弹窗提示"需要人类介入"，显示3次封驳原因汇总
- 伏笔任务未完成：伏笔操作区红色边框+提示文字
- 场景未保存离开：浏览器弹出确认提示
- 场景内容超长：编辑区可全屏模式
- 定稿后修改：弹窗提示"已定稿，修改将创建新版本"
```
