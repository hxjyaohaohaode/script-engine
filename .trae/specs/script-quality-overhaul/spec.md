# 互动影游剧本质量全面重构 — 对标行业标准的剧本生成系统

> **日期**: 2026-05-10
> **基于**: 行业深度调研 + 系统现状分析 + 用户反馈
> **范围**: 世界观同步、角色关系网络、伏笔体系、章节结构与互动选择、完整剧本生成

---

## Why

当前系统生成的剧本存在五大核心缺陷，导致输出完全不可用于互动影游制作：

1. **世界观内容不同步**：流水线生成的世界观内容未写入前端div展示的配置项，导致用户看到的是空内容
2. **角色关系网络缺失**：角色之间联系太少，且关系图谱未在界面中充分显现，无法支撑"低分支高情感"的叙事范式
3. **伏笔体系劣质**：生成内容不全，缺少哇塞时刻、联系弧线，伏笔间逻辑性和适配性差，与世界观/人物/场景关联性极差
4. **章节结构单一**：每章仅一节内容，内容刻板浅薄，未关联世界观/角色/伏笔/场景/时间线，AI未自动设置节与合理分配字数
5. **剧本不完整**：生成的不是完整剧本，缺少互动选择、分支设计、后果链等互动影游核心要素

行业调研表明：互动影游剧本必须具备Agency（代理感）、Branching（分支）、Consequence（后果）三大支柱，核心伏笔回收率不低于80%，每章需2-5个互动节点，分支层级控制在3-4级以内。当前系统远未达到这些标准。

## What Changes

### 世界观数据流修复
- **BREAKING**: 流水线Phase 0生成的世界观内容必须同步写入`project_config`表的`custom_checker_rules.world_settings`字段
- 前端WorldSettings页面的`fetchConfigs`必须从`project_config`表读取已同步的数据
- 流水线`world_builder`步骤的输出必须触发`DataEvents.WORLD_CONFIG_UPDATED`事件

### 角色关系网络升级
- 角色生成必须基于世界观上下文，生成"好感度-信任度-利益关联"三维关系模型
- 关系网络图谱增加"关系弧线"可视化：显示关系变化趋势、引爆点标记、暗线关系
- 角色间必须生成至少N*(N-1)/2 * 0.6条关系（60%覆盖率），确保网络密度
- 新增"关系弧线"数据模型：记录关系值随场景的变化轨迹

### 伏笔体系全面重构
- 伏笔生成必须从核心真相反推，生成三层伏笔体系（全剧级5-8条/章节级20-30条/场景级60-100条）
- 每条伏笔必须包含三层含义（表面/深层/真相）+ 哇塞方案 + 关联弧线
- 伏笔间必须建立DEPENDS_ON/SUPPORTS/ENABLES/CONFLICTS_WITH四类关联
- 伏笔必须与世界观、角色、场景建立双向引用关联
- 新增"联系弧线"可视化：伏笔→角色→场景→世界观的全链路关联图

### 章节结构与互动选择重构
- **BREAKING**: 每章必须由AI自动设置2-5个"节"（Section），每节对应一个互动节点
- 章节生成必须关联世界观、角色、伏笔、场景、时间线
- 每个互动节点必须包含：2-3个差异化选择 + 后果链推演 + 隐藏选项条件
- 章节字数必须合理分配：根据情感目标和伏笔任务自动规划
- 新增"节"数据模型：section_number / title / word_target / scene_ids / choices / emotion_target

### 完整剧本生成
- 剧本必须包含完整的互动要素：选择节点、分支路径、后果链、角色状态变化、伏笔操作
- 场景生成必须包含：叙述+对白（含潜台词）+动作+互动选择+伏笔操作+因果链
- 新增"剧本预览"功能：以互动影游格式展示完整剧本（选择→分支→后果）
- 导出格式必须包含互动标记（选择点、分支标记、跳转指令）

## ADDED Requirements

### Requirement: 世界观数据流同步
系统 SHALL 确保流水线生成的世界观内容与前端展示完全同步。

#### Scenario: 流水线生成世界观后前端可见
- **GIVEN** 用户通过流水线Phase 0生成世界观
- **WHEN** world_builder步骤完成
- **THEN** 生成的内容必须写入`project_config`表的`custom_checker_rules.world_settings`字段
- **AND** 前端WorldSettings页面刷新后能看到所有已生成的配置项内容
- **AND** 每个配置项的锁定状态正确反映

#### Scenario: 世界观AI生成方案与已有内容一致
- **GIVEN** 某配置项已有流水线生成的内容
- **WHEN** 用户点击"AI生成方案"
- **THEN** AI必须基于已有内容进行扩展/优化，而非生成全新无关内容
- **AND** 方案必须与其他已锁定设定保持逻辑自洽

### Requirement: 角色关系网络密度与可视化
系统 SHALL 生成高密度角色关系网络，并提供关系弧线可视化。

#### Scenario: 角色关系网络密度达标
- **GIVEN** 项目有N个活跃角色
- **WHEN** AI生成关系网络
- **THEN** 关系数量不低于N*(N-1)/2 * 0.6（60%覆盖率）
- **AND** 每条关系必须包含：关系类型 + 信任度 + 好感度 + 信息不对称（A知道B什么/B知道A什么）
- **AND** 关系类型必须覆盖：亲属/恋人/挚友/宿敌/师徒/暗线（秘密盟友/隐藏仇敌）等

#### Scenario: 关系弧线可视化
- **GIVEN** 角色间存在关系且场景已生成
- **WHEN** 用户查看角色关系图谱
- **THEN** 连线上显示关系变化趋势（↑改善/↓恶化/→稳定）
- **AND** 引爆点（trust骤降>30）用红色爆炸图标标记
- **AND** 暗线关系用紫色虚线+锁图标标记（需特定场景触发后才可见）
- **AND** 点击连线可查看完整关系弧线时间轴

#### Scenario: 角色生成基于世界观上下文
- **GIVEN** 世界观已设定（核心矛盾/社会结构/科技魔法体系等）
- **WHEN** AI生成角色
- **THEN** 每个角色的核心动机必须与世界观核心矛盾直接关联
- **AND** 角色的暗秘密必须与世界观的"深层/核心"设定关联
- **AND** 角色的行为约束必须与世界观的"约束条件/不可能事项"一致

### Requirement: 伏笔体系三层架构与关联弧线
系统 SHALL 生成符合行业标准的三层伏笔体系，并建立与世界观/角色/场景的完整关联。

#### Scenario: 三层伏笔体系生成
- **GIVEN** 世界观已设定且核心真相已定义
- **WHEN** AI设计伏笔体系
- **THEN** 生成全剧级伏笔5-8条（每条含三层含义+哇塞方案+埋设/强化/揭露位置）
- **AND** 生成章节级伏笔20-30条（每条含至少两层含义+与全剧级伏笔的关联）
- **AND** 生成场景级伏笔60-100条（每条含表面层+与章节级伏笔的关联）
- **AND** 伏笔间建立DEPENDS_ON/SUPPORTS/ENABLES/CONFLICTS_WITH四类关联
- **AND** 核心伏笔回收率规划不低于80%

#### Scenario: 伏笔与世界观/角色/场景关联
- **GIVEN** 伏笔已生成
- **WHEN** 用户查看伏笔详情
- **THEN** 可看到该伏笔关联的世界观设定（如：此伏笔揭露了"科技体系"的深层规则）
- **AND** 可看到该伏笔关联的角色（如：此伏笔涉及角色A的暗秘密）
- **AND** 可看到该伏笔关联的场景（如：此伏笔在第3章第2节被埋设）
- **AND** 可看到该伏笔的"联系弧线"：从世界观设定→角色动机→伏笔埋设→强化→揭露的完整路径

#### Scenario: 哇塞方案设计
- **GIVEN** 全剧级伏笔已定义
- **WHEN** AI设计哇塞方案
- **THEN** 对每条全剧级伏笔尝试5种创意类型（反转/信息差/角色弧光/世界观颠覆/情感核弹）
- **AND** 每个方案4维评分：可预测性(3-7)/情感冲击(≥8)/逻辑自洽(≥8)/回望价值(≥7)
- **AND** 每条伏笔保留2-3个最优方案供用户选择
- **AND** 哇塞方案必须与世界观核心真相直接关联

### Requirement: 章节结构与互动选择设计
系统 SHALL 为每章自动设置2-5个"节"，每节包含完整的互动选择设计。

#### Scenario: 章节自动分节
- **GIVEN** 章节大纲已生成
- **WHEN** AI展开章节内容
- **THEN** 每章自动设置2-5个"节"（Section），根据情感目标和伏笔任务合理分配
- **AND** 每节包含：标题/目标字数/情感目标/涉及角色/伏笔任务/互动选择
- **AND** 章节总字数根据项目目标字数和章节数自动分配
- **AND** 节的排列遵循"漏斗模型"：开头自由探索→中段关键抉择→结尾汇入主线

#### Scenario: 互动选择设计
- **GIVEN** 章节节的互动节点
- **WHEN** AI设计互动选择
- **THEN** 每个互动节点包含2-3个差异化选择（不同代价，非对/错）
- **AND** 每个选择包含：选项文本/直接后果/间接后果/远期后果/角色关系影响
- **AND** 关键节点包含1个隐藏选项（需特定前置条件触发）
- **AND** 选择设计遵循"道德灰度"原则：没有完美答案，每个选择都有代价
- **AND** 分支层级控制在3-4级以内，避免剧情过度分散

#### Scenario: 章节内容关联世界观/角色/伏笔
- **GIVEN** 世界观/角色/伏笔已设定
- **WHEN** AI生成章节内容
- **THEN** 章节摘要必须引用世界观设定（如：本章揭露了"社会结构"的深层规则）
- **AND** 章节必须标注聚焦角色及其关系变化预期
- **AND** 章节必须标注伏笔任务（哪些伏笔需在此章埋设/强化/回收）
- **AND** 章节情感目标必须与全局情感曲线一致

### Requirement: 完整剧本生成与互动要素
系统 SHALL 生成包含完整互动要素的剧本，可直接用于互动影游制作。

#### Scenario: 场景生成包含完整互动要素
- **GIVEN** 场景创作任务已派发
- **WHEN** 创作Agent生成场景
- **THEN** 场景必须包含：叙述（画面感+感官描写）+ 对白（含潜台词）+ 动作 + 互动选择 + 伏笔操作 + 因果链
- **AND** 对白必须反映角色语言风格和口头禅
- **AND** 互动选择必须包含2-3个选项 + 后果推演
- **AND** 伏笔操作必须标注操作类型（plant/reinforce/reveal）和涉及的伏笔ID
- **AND** 因果链必须包含：前置条件→催化剂→直接结果→间接结果→远期结果

#### Scenario: 剧本预览以互动影游格式展示
- **GIVEN** 项目有已定稿的场景
- **WHEN** 用户打开剧本预览
- **THEN** 以互动影游格式展示：场景叙述→对白→选择点（高亮标记）→分支路径→后果
- **AND** 选择点可点击查看各选项的后果链
- **AND** 伏笔操作用特殊标记显示（🌱埋设/🔄强化/💡回收）
- **AND** 哇塞时刻用★标记
- **AND** 支持按章节/角色/伏笔筛选查看

#### Scenario: 导出包含互动标记
- **GIVEN** 用户选择导出剧本
- **WHEN** 选择Markdown或JSON格式
- **THEN** 导出内容包含互动标记：[CHOICE]选择点 / [BRANCH]分支 / [CONSEQUENCE]后果 / [FORESHADOW]伏笔操作
- **AND** JSON格式包含完整的结构化互动数据（choices/branches/consequences/foreshadow_ops）
- **AND** Markdown格式以可读的互动剧本格式呈现

## MODIFIED Requirements

### Requirement: 世界观AI生成方案升级
原AI生成方案仅返回3个简短文本。**修改为**：AI生成方案必须基于已有世界观上下文，每个方案1000-3000字，包含具体细节和画面感，且与其他已锁定设定保持逻辑自洽。

### Requirement: 章节大纲AI展开升级
原AI展开大纲仅生成章节标题和摘要。**修改为**：AI展开大纲必须生成完整的章节结构，包含2-5个节、互动选择设计、伏笔任务分配、角色聚焦、情感目标，并与世界观/角色/伏笔深度关联。

### Requirement: 伏笔AI设计升级
原AI设计伏笔仅生成简单伏笔列表。**修改为**：AI设计伏笔必须从核心真相反推，生成三层伏笔体系（全剧级/章节级/场景级），每条伏笔包含三层含义+哇塞方案+关联弧线，伏笔间建立四类关联。

## REMOVED Requirements

无移除的需求。本次为增量升级和质量重构，不删除任何已有功能。
